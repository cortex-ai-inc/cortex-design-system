Editor Configuration

The rich text editor is configured with a curated set of extensions. The StarterKit is customized to expose only h2 and h3 for headings — h1, h4, h5, h6 are excluded to maintain content hierarchy.

Extensions
Extension Config Notes
StarterKit heading: { levels: [2, 3] } h2 and h3 only — no h1/h4/h5/h6
Image inline: false, allowBase64: true Block-level images, base64 data URLs
Link openOnClick: true Auto-link detection
Placeholder placeholder: "Write something..." Empty state prompt

SSR Handling

The rich text editor cannot render on the server because TipTap requires the DOM. During SSR, a static placeholder div is rendered. The editor mounts client-side only, wrapped in a useEffect or conditional render based on typeof window !== "undefined".

Paste Handler

Images pasted from the clipboard are intercepted, converted to base64 data URLs, and inserted as block-level images. The paste handler enforces a 10 MB size limit.

Paste Handling Flow
1 User pastes image from clipboard
2 Paste handler checks file size ≤ 10 MB
3 FileReader converts to base64 data URL
4 Image inserted into editor via chain().setImage().run()
Maximum size: 10 MB. Images are block-level (not inline) with base64 encoding.
Important: Base64 images can significantly increase content size. A 1 MB image becomes approximately 1.37 MB as base64. Use sparingly and consider server-side upload for production use cases.

Rendering

Stored rich text HTML is rendered using isomorphic-dompurify for XSS prevention. The RichTextRenderer component sanitizes the HTML and renders it safely.
