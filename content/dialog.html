Dialog (Modal)

Modal overlay for confirmations, forms, and focused tasks that require user attention.

React Radix Dialog

Overview

Dialogs interrupt the current flow to present critical information or request user input. Built on Radix Dialog, they provide accessible modal behavior: focus trapping, escape to close, click-outside to close, and proper ARIA attributes. An overlay backdrop with bg-black/60 backdrop-blur-sm sits at z-[200] to obscure the page beneath.

Content panels use bg-surface-container-low with a soft shadow (shadow-ambient) and enter via animate-slide-up. Width is determined by a tiered system depending on the dialog's purpose.

Width Tiers

Width Pixels Use Case
sm 440px Confirmations, alerts, short prompts
md 560px Forms, item creation, settings
lg 720px Complex forms, multi-step workflows

Structure

Component Role
Dialog Root context provider for open/close state
DialogTrigger Button or element that opens the dialog
DialogContent The modal panel with overlay + positioning
DialogHeader Top section with title + description
DialogTitle Accessible title (announced on open)
DialogDescription Optional description text
DialogFooter Bottom section for action buttons
DialogClose Button that closes the dialog

AlertDialog Variant

For destructive confirmations (delete, remove, irreversible actions), use AlertDialog instead of regular Dialog. AlertDialog does not close on click-outside, forcing explicit user choice. The action button uses the destructive variant.

Usage Guidelines

Do

  • Use Dialog for forms that benefit from context preservation
  • Use AlertDialog for destructive confirmations
  • Set aria-describedby linking to description
  • Focus the first focusable element on open

Don't

  • Stack multiple dialogs — use in-page alternatives
  • Use for complex multi-step flows — prefer dedicated pages
  • Remove the overlay or backdrop blur
