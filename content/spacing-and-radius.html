Spacing & Radius

Consistent spacing and border radius based on a --radius: 0.5rem base.

Border Radius

TokenValueExample
--radius (base) 0.5rem (8px)
xl calc(var(--radius) + 4px) = 12px
lg var(--radius) = 8px
md calc(var(--radius) - 2px) = 6px
sm calc(var(--radius) - 4px) = 4px
xs calc(var(--radius) - 6px) = 2px

In practice, rounded-sm (4px) is used most commonly for buttons, inputs, and cards.

Spacing

Uses Tailwind's default spacing scale. Common values in practice:

ClassValueTypical Usage
p-28pxCompact padding for badges, tags, small elements
p-312pxInput padding, small card padding
p-416pxStandard card padding, modal content
p-520pxGenerous card padding, page sections
p-624pxPage-level padding, wide containers
gap-28pxTight element spacing (icon + label)
gap-312pxDefault flex/grid gap
gap-416pxSection spacing, card grids
gap-624pxLarge section spacing

Constraint

⚠️ Never exceed 0.375rem (6px)

In cortex-coder-front, border-radius is constrained to maximum 6px in practice, overriding the shadcn defaults.
