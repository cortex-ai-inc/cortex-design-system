Badge

Small label for status indicators, counts, and metadata tags.

React

Overview

Badges provide a compact way to display status, counts, or categorical labels. They render as inline-flex elements with uppercase text (11px), rounded-sm (4px) corners, and horizontal padding. Unlike tags, badges carry semantic meaning through color.

Variants

Variant Class Visual Usage
default badge badge-default Surface-container-high bg, muted text Generic label, neutral status
primary badge badge-primary Blue-toned bg, k-primary text Primary label, "active" status
success badge badge-success Green-toned bg, k-secondary text "Completed", "approved", "online"
warning badge badge-warning Yellow-toned bg, k-warning text "Pending", "in progress", "awaiting"
destructive badge badge-destructive Red-toned bg, k-error text "Error", "blocked", "overdue"
outline badge badge-outline Transparent bg, border only Subtle labels, secondary info

Usage Guidelines

Do

  • Use semantic variants matching the status meaning
  • Keep badge text short — one or two words
  • Use uppercase text for consistency

Don't

  • Use multiple colors for the same status type
  • Put interactive elements inside badges
  • Use badges for long-form text
