Shadows

Minimal shadow usage — prefer tonal layering over drop shadows.

Elevation Tiers

Tonal layering is the primary depth language — cards and panels get no shadow. Four tiers exist for elements that float above the page:

TierTokenValueElements
0 none — tonal only — Cards, panels, page sections
1 shadow-xs 0 1px 2px 0 rgb(0 0 0 / 0.05) Switch thumb, tiny raised details
2 shadow-overlay 0 8px 24px rgba(0, 0, 0, 0.40) Dropdowns, selects, tooltips, popovers, command palette
3 shadow-ambient 0 32px 64px rgba(0, 0, 0, 0.12) Modals, dialogs, sheets

⚠️ Glow rules

Only on gradient-primary elements; at most one per viewport. Allowed on hero/marketing CTAs, empty-state primary actions, onboarding finish steps. Never on in-app toolbar/form buttons, inputs, or cards.

When to Use Shadows

Shadows are reserved exclusively for floating elements. Small floats (dropdowns, tooltips, popovers, command) use shadow-overlay; full modal surfaces use shadow-ambient. Cards and panels use tonal layering instead.

💡 Design Principle

Surface colors create depth: surface-dim → surface-container-lowest → surface-container-low → surface-container-high. Each level sits "higher" than the last without requiring a drop shadow.
