EmptyState

A structured empty state component for displaying placeholder content when lists, searches, or views have no data to show.

Media variant with custom illustration placeholder.

The EmptyState component provides a consistent layout for empty views. It includes sub-components: EmptyHeader, EmptyTitle, EmptyDescription, EmptyContent, and EmptyMedia. The media variant can display as either an icon (in a size-10 container with bg-muted) or a custom illustration.

API

Component Props Description
Empty className?: string, children: ReactNode Root container for empty state.
EmptyMedia variant?: 'default' | 'icon', children?: ReactNode Media/illustration area. 'icon' variant renders children in a size-10 container.
EmptyTitle className?: string, children: ReactNode Empty state heading.
EmptyDescription className?: string, children: ReactNode Descriptive text explaining the empty state.
EmptyContent className?: string, children: ReactNode Action area (buttons, links, etc.).

Usage

{`import { Empty, EmptyMedia, EmptyTitle, EmptyDescription, EmptyContent } from '@/components/ui/empty-state'

// Basic empty state
<Empty>
  <EmptyMedia variant="icon">
    <FileIcon />
  </EmptyMedia>
  <EmptyTitle>No tickets found</EmptyTitle>
  <EmptyDescription>There are no tickets matching your criteria.</EmptyDescription>
  <EmptyContent>
    <Button>Create Ticket</Button>
  </EmptyContent>
</Empty>

// Check in list rendering:
{tickets.length === 0 ? (
  <Empty>
    <EmptyTitle>No results</EmptyTitle>
    <EmptyDescription>Try adjusting your search.</EmptyDescription>
  </Empty>
) : (
  tickets.map(ticket => ...)
)}`}
