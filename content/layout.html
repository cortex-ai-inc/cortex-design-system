App Shell

The application shell consists of a persistent sidebar on the left, a topbar at the top, and a main content area. The sidebar provides primary navigation, while the topbar contains secondary navigation, breadcrumbs, and user controls.

Layout Structure
Sidebar (72px collapsed / 260px expanded) + Topbar (48px) + Main content area

Sidebar Dimensions

The sidebar has two states: expanded (260px) and collapsed (72px). On desktop the sidebar is always visible; on mobile it renders as an overlay panel.

Expanded
width: 260px

Shows icon + label for all nav items. Default state on desktop.

Collapsed
width: 72px

Icons only. Labels hidden. Toggle via hamburger menu or keyboard shortcut.

Z-Index Layers

A clear z-index hierarchy prevents stacking conflicts across the application.

Layer Value Usage
base 0 Page content, cards, grids
sidebar 10 Sidebar navigation panel
topbar 20 Top bar, sticky headers
dropdown 100 Dropdown menus, popovers, tooltips
modal 200 Dialogs, modals, slide-overs
toast 300 Toast notifications (sonner)

Page Headers

Content Width Caps

Content areas use max-width constraints to maintain readability and prevent overly wide layouts.

Container Class Use Case
Default max-w-7xl Most list pages, dashboards
Narrow max-w-5xl Forms, detail pages, settings
Editor max-w-6xl Rich text editor, content creation
Rule: Always apply width caps to the .page-body or an inner wrapper. Never constrain individual sections within the page — use the cap once at the top level.

Responsive Behavior

Desktop
  • Sidebar always visible (expanded or collapsed)
  • Topbar fixed at top
  • Content scrolls independently
  • Column layouts use md:grid-cols-2
  • Headers: sm:flex-row
Mobile
  • Sidebar hidden by default, overlay on toggle
  • Topbar contains hamburger menu + breadcrumb
  • Single column layout
  • Headers stack vertically
  • Tables scroll horizontally

Spacing Conventions

Consistent spacing creates visual rhythm. The system uses a 4px base unit with the following common gaps:

Token Tailwind Pixel
xs gap-1 4px
sm gap-2 8px
md gap-3 12px
lg gap-4 16px
xl gap-6 24px
Important: The sidebar uses gap-4 between nav items. Forms use gap-x-6 gap-y-5. Lists use gap-3 between columns. Do not introduce ad-hoc spacing values.
