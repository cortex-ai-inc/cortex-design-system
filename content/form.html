Form & Field

Structured form layout patterns with react-hook-form integration and consistent field components.

React react-hook-form Zod

Overview

Forms follow a consistent pattern using react-hook-form with @hookform/resolvers/zod for validation. The system provides a Field component with configurable orientation and a FormSection pattern for grouping related fields into card-like areas with icons and titles.

Standard form width is capped at max-w-5xl. Two-column layouts use grid grid-cols-1 md:grid-cols-2 gap-x-6 gap-y-5.

FormSection Pattern

FormSection groups related fields into a card-like container with an icon, title, and optional description. Each section represents a logical group within the form (e.g., "Profile Details", "Notifications").

Usage Guidelines

Do

  • Use FormSection to group related fields logically
  • Cap form width at max-w-5xl
  • Use two-column grid for field pairs (label + value)
  • Show validation errors inline below each field
  • Disable submit button while form is submitting

Don't

  • Place more than two columns in a grid row
  • Use horizontal orientation for long labels
  • Show success toasts on every save — use subtle indicators
  • Reset form state unexpectedly on background refetches
