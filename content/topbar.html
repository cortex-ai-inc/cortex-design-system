TopBar

Top navigation bar with breadcrumbs, search, and user actions.

React

Overview

The TopBar sits at the top of authenticated application views, spanning the full viewport width with a fixed height of h-12 (48px). It uses bg-surface-dim (#111319) with a bottom border of border-b ghost-border and sticky top-0 positioning.

The TopBar is divided into two zones: the left side contains the hamburger menu (mobile only) and breadcrumb navigation. The right side contains the notification bell and user avatar dropdown.

Breadcrumbs

Breadcrumbs show the current page location within the route hierarchy. Each segment is separated by a forward slash (/) with muted styling. The final segment (current page) is highlighted with font-medium weight and on-surface color.

Route Mapping

Route Breadcrumb
/ Dashboard
/tickets Dashboard / Tickets
/tickets/new Dashboard / Tickets / New Ticket
/tickets/{id} Dashboard / Tickets / {title or id}
/profile Dashboard / Profile
/admin Admin
/admin/tickets Admin / Tickets
/admin/settings Admin / Settings

Notification Bell

The notification bell is a btn-ghost btn-icon button that displays an unread indicator dot when there are unseen notifications. The bell polls every 30 seconds via React Query (refetchInterval: 30000) and updates the badge based on the last-check timestamp stored in localStorage (cortex.notifications.since).

Usage Guidelines

Do

  • Show breadcrumbs reflecting the full route hierarchy
  • Use the notification bell with polling-based unread indicator
  • Show user avatar with dropdown for profile/account actions
  • Keep the topbar sticky at all times

Don't

  • Place navigation links in the topbar — use sidebar
  • Overcrowd the topbar — limit to 3-4 right-side actions
  • Show the hamburger menu on desktop breakpoints
