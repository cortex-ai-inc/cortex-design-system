FileUpload

Drag-and-drop file input with a click-to-browse fallback. Accepted files render as Item rows with per-file progress, success, and error states; rejected files surface in an error alert.

New React

Overview

FileUpload wraps a visually hidden native <input type="file"> and HTML5 drag events — no library required (react-dropzone is an optional convenience, not a dependency). The dropzone is keyboard-accessible (role="button", Enter/Space opens the dialog) and announces its constraints via aria-describedby. The file list composes ItemGroup / Item variant="outline" size="sm", embedding Progress while uploading. See file-upload.md for the full spec.

File list

Each accepted file is an Item variant="outline" size="sm" row: ItemMedia variant="icon" on the left, filename as ItemTitle, size/status as ItemDescription, and ghost icon buttons in ItemActions. Uploading rows swap the description for a Progress bar (h-1.5) with a percent label.

Type rejection

Files that fail the accept / maxSize / maxFiles constraints never enter the list. Surface them in a destructive Alert directly below the dropzone, naming the file and the violated constraint.

Props

Prop Type Default Description
accept string — Accepted MIME types / extensions, forwarded to the hidden input
maxSize number — Max file size in bytes; larger files are rejected
maxFiles number — Max number of files; extras are rejected
multiple boolean false Allow multi-select and multi-drop
disabled boolean false Disables the dropzone and hidden input
onFilesAccepted (files: File[]) => void — Called with files that pass all constraints
onFileRejected (file: File, reason: "type" | "size" | "count") => void — Called per rejected file; drives the rejection alert
onRemove (file: File) => void — Called when a row's remove action is clicked
onRetry (file: File) => void — Called when a failed row's retry action is clicked

Usage Guidelines

Do

  • State accepted types and max size upfront in the dropzone hint
  • List every accepted file as an Item row with remove/retry actions
  • Offer retry on failed rows instead of forcing a re-drop
  • Keep the dropzone clickable and keyboard-operable (Enter/Space)

Don't

  • Auto-upload silently without rendering the file list
  • Put upload flows inside modals — use dedicated pages or sections
  • Rely on drag-and-drop as the only way in
  • Swallow rejections — name the file and the violated constraint
