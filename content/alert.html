Alert

Persistent inline message for contextual status — stays in the page until the condition is resolved.

New React

Overview

Alert is a custom component (no Radix) — a plain div role="alert" styled with CVA. It renders a tinted background at 8% opacity with a 3px left accent border, an optional lucide icon (w-4 h-4, strokeWidth={1.5}), a semibold title, and a muted description. Non-interactive: no hover or focus states on the alert itself.

Choose the right messaging component: Alert is persistent contextual messaging (form-level errors, degraded-state banners) · Toast is transient feedback after an action · AlertDialog is a blocking confirmation.

Usage Guidelines

Do

  • Use for form-level errors and persistent contextual messaging
  • Use for degraded-state banners at the top of the affected section
  • Match variant to semantic meaning — green success, red error, yellow warning, blue info

Don't

  • Use for transient feedback — use Toast instead
  • Use for confirmations — use AlertDialog instead
  • Stack more than one Alert per section — keep only the most severe
