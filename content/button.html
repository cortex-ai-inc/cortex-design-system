Button

Triggers an action or navigates to a destination. The most frequently used interactive element in the system.

React Radix Slot

Overview

Buttons communicate actions users can take. The system provides six visual variants, four size options, and supports an asChild pattern via Radix Slot for composing with framework routers or custom elements. All buttons use rounded-sm (4px), display inline-flex items-center justify-center, and inherit font-medium text styling.

Size Variants

Icon Buttons

Icon buttons use the btn-icon size class to create a square button (w-8 h-8) that contains only an icon. They are commonly used in toolbars, table row actions, and dialog close buttons. Always provide an aria-label for accessibility.

States

Hover

Buttons darken slightly on hover. Primary becomes brighter on the gradient. Ghost and outline fill with bg-surface-container-high.

Focus

Focus-visible shows a ring-2 ring-k-primary ring offset by 2px. Never remove focus indicators.

Disabled

Disabled buttons use opacity-50 and pointer-events-none. Do not use disabled buttons without a clear reason — prefer keeping them visible but explaining why the action is unavailable.

Usage Guidelines

Do

  • Use one primary button per section
  • Use destructive variant for delete/remove actions
  • Provide aria-label on icon-only buttons
  • Use asChild when wrapping router links

Don't

  • Stack multiple primary buttons — use one primary, rest secondary
  • Use ghost buttons for critical actions
  • Remove focus indicators for custom styling
  • Nest buttons inside each other
