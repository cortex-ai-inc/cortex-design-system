Card

Contained surface for grouping related content and actions.

React

Overview

Cards are the primary container pattern for grouped content. They sit on the surface layer using bg-surface-container-low with a ghost-border stroke, creating a subtle distinction from the page background. The Card component (available in cortex-coder-front) provides seven sub-components for structured layouts.

Note: The Card component is used in cortex-coder-front. The cortex-support-front project does not have a Card component and surfaces content directly.

Structure

Component Role
Card Root wrapper with surface background + border + rounded-sm
CardHeader Top section containing title, description, and actions
CardTitle Card heading using text-title-sm
CardDescription Subtitle text using on-surface-variant color
CardAction Action slot in the top-right of CardHeader
CardContent Main body area with padding
CardFooter Bottom section for actions (buttons, links)

  </CardFooter>

Styling
  • Background: bg-surface-container-low (#191B22)
  • Border: ghost-border (1px semi-transparent)
  • Radius: rounded-sm (4px)
  • Shadow: none (tonal layering provides depth)
  • Padding: p-4 in header/content, pt-0 in footer

Usage Guidelines

Do

  • Use cards to group related content with clear hierarchy
  • Use CardAction for contextual menu or tertiary actions
  • Keep card content focused and scannable

Don't

  • Nest cards inside cards
  • Place primary CTAs in CardFooter — use page-level actions
  • Overcrowd — if content exceeds 400px height, consider a different layout
