Combobox

Searchable select composed from Popover + Command. Type to filter, pick one value — or many.

New React Radix Popover cmdk

Overview

The Combobox is a composition recipe, not a new primitive: a cmdk Command rendered inside a Radix Popover, opened from a Button styled as a form field. The trigger copies the Select trigger tokens verbatim (h-9, ghost-border-20, bg-surface-dim, rounded-sm, focus:ring-2 focus:ring-k-primary-container/50) and swaps the chevron for ChevronsUpDown to signal searchability. The panel uses the shared popover recipe (bg-surface-container-low, shadow-overlay, z-[100]) sized to the trigger via w-[--radix-popover-trigger-width] p-0.

MultiSelect

The multiple-selection variant swaps the trigger height for min-h-9 h-auto flex-wrap gap-1 py-1.5 and renders one Badge chip (bg-surface-container-high text-on-surface-variant rounded-sm, icons [&_svg]:size-3) per selected value, plus a +N more overflow chip. Items toggle on select and the popover stays open; the chip's X removes a value without opening the panel (stop propagation).

Usage Guidelines

Do

  • Use for searchable lists with more than ~7 options (users, projects, tags)
  • Use for async lookups — debounce input and show the spinner row while fetching
  • Use MultiSelect with badge chips for multi-tag pickers; keep the popover open on toggle
  • Keep the trigger h-9 (min-h-9 when multiple) so it aligns with Inputs and Selects
  • Size the panel with w-[--radix-popover-trigger-width] p-0

Don't

  • Use for fewer than ~7 static options — use Select instead
  • Use for global actions or navigation — that's the CommandDialog (Cmd+K)
  • Build a new primitive — compose Popover + Command
  • Fill the selected row blue — selection is the trailing Check in k-primary-container
  • Add a red error border to the trigger — errors go through FormMessage
