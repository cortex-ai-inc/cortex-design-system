Forms

Form layout, field orientation, validation, and submission patterns. Based on react-hook-form + Zod v4.

TypeScript React 19 Zod v4
Rule: Create/edit forms always use dedicated pages, never modals or dialogs. Detail pages toggle between view and edit modes via the ?mode=edit query parameter.

Form Layout

Forms use a responsive grid layout. On mobile, fields stack in a single column. On desktop (md:), fields split into two columns. The form width is capped at max-w-5xl for readability.

Field Orientation

Fields can use either vertical (default, label above input) or horizontal (label beside input) orientation. Vertical is preferred for most forms. Horizontal is used for compact layouts like settings panels.

Vertical (Default)

Error Handling

Validation errors appear below the field in red. The error state is triggered by Zod validation via react-hook-form's formState.errors.

Submit Button Alignment

Form action buttons are aligned to the right to follow the reading flow. Use flex justify-end gap-2 for the button group.

.flex.justify-end.gap-2 — Cancel (ghost) then Save (primary)

View/Edit Toggle

Detail pages support both viewing and editing at the same URL. Switching between modes is done via the ?mode=edit query parameter — never via a separate route.

/admin/tickets/123 View mode
/admin/tickets/123?mode=edit Edit mode
?mode=edit query parameter toggles between read and edit states

Validation Patterns

Forms use Zod v4 schemas for validation with @hookform/resolvers/zod. Standard patterns include:

Pattern Zod Schema Example
Required z.string().min(1) Name, title fields
Email z.string().email() Login, invite forms
Optional z.string().optional() Description, notes
Enum z.enum([...]) Status, priority selects
Number z.coerce.number() Numeric inputs from strings
